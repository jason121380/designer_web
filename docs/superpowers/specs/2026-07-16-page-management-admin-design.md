# 頁面管理後台設計

## 目標

將既有文章 CMS 後台改為單一用途的一頁式網站管理後台。管理者登入後直接進入「頁面管理」，並能依照前台顯示順序修改每個區塊。

## 導覽與路由

- 側邊選單只保留「頁面管理」。
- 登入成功、`/admin` 與舊的 `/admin/dashboard` 都導向 `/admin/page-management`。
- 文章、分類、標籤、媒體庫、流量分析、用戶管理與工程工具不再出現在介面。
- 舊後台功能網址統一導回 `/admin/page-management`，避免保留可操作入口。
- 側邊欄底部保留目前登入帳號與登出按鈕。

## 頁面管理

編輯器依前台順序分成以下區塊：

1. 品牌與導覽
2. 首屏形象
3. 活動 DM
4. 接髮介紹
5. 其他服務
6. 作品影片
7. 分期資訊
8. 價目表
9. 環境介紹
10. 聯絡資訊

每個區塊可編輯文字、連結與適用的圖片或影片 URL。圖片與影片先經 Cloudflare 或既有上傳 API 上傳，再將回傳 URL 寫入頁面設定。空的媒體區塊不在前台顯示。

## 資料流

- 所有前台設定儲存在 PostgreSQL 的 `designer_web_content` 設定資料中。
- 前台與後台使用同一份結構化資料，避免後台修改後前台未更新。
- 儲存時進行欄位正規化，空白項目不輸出到前台。

## 品牌

- 後台使用純文字 `Designer Web`，不顯示舊 Logo。
- 移除使用者可見的 `mifaso` 與「迷髮所」字樣。

## 驗證

- 測試側邊欄只剩頁面管理。
- 測試登入與舊後台路由會導向頁面管理。
- 測試每個前台區塊都有對應設定欄位並可正規化儲存。
- 執行 TypeScript、建置與本機桌機／手機瀏覽器驗證。
